A2UI (Agent-to-User Interface) is an open-source protocol and renderer ecosystem that enables AI agents to generate native UI across any platform. Agents produce declarative JSON describing what to show; your client renders it using its own trusted components — no code execution, no iframes, no security risk.

## Why A2UI

Text-only agent interactions require multiple round-trips. A form that collects a restaurant booking takes five exchanges in chat — one well-designed UI collapses it to a single interaction. A2UI gives agents the ability to generate that UI without the security risk of running arbitrary code on your client.

<CardGroup cols={3}>
  <Card title="Security First" icon="shield-check">
    Declarative JSON, not executable code. Your client's component catalog is the allowlist — agents can only request pre-approved components.
  </Card>
  <Card title="Framework Agnostic" icon="layer-group">
    One A2UI payload renders on React, Angular, Lit, and Flutter. The same agent response works everywhere.
  </Card>
  <Card title="LLM Friendly" icon="microchip-ai">
    Flat adjacency-list structure is easy for LLMs to generate incrementally and stream progressively.
  </Card>
</CardGroup>

## How It Works

<Steps>
  <Step title="Agent generates UI">
    Your AI agent (using Gemini, GPT-4, or any model) produces a sequence of A2UI JSON messages describing the component tree and data model.
  </Step>
  <Step title="Messages travel over transport">
    The JSON stream is sent to your client via A2A Protocol, AG-UI (CopilotKit), WebSockets, or SSE.
  </Step>
  <Step title="Renderer parses and resolves">
    Your client's A2UI renderer (React, Angular, Lit, or Flutter) processes the messages and looks up each component in your trusted catalog.
  </Step>
  <Step title="Native UI renders">
    Components render using your app's own widgets, inheriting your design system, accessibility settings, and theming — no iframes or shadow DOM.
  </Step>
</Steps>
